# Tempo — 비즈니스PT 레드센터 통합 운영 플랫폼 사이트맵

## 시스템 구조 개요

Tempo는 **단일 SaaS 앱**입니다. 네이버 카페처럼 "사본 사이트"를 만드는 구조가 아니라, 하나의 앱 안에서 역할(tempoRole)에 따라 접근 가능한 기능이 달라집니다.

```
[Manus OAuth 로그인] → 신규 가입자는 tempoRole=trainer로 시작
                    → OWNER_OPEN_ID와 일치하면 tempoRole=owner 자동 부여
                    → 초대 링크(/join?token=...)로 가입 시 초대된 역할 자동 부여
```

---

## 역할 계층 구조

| 역할 | 한국어 | 권한 범위 |
|------|--------|-----------|
| `owner` | 대표 | 모든 기능 + 시스템 설정 |
| `center_manager` | 책임센터장 | 관리자 기능 + 어드민 연동 |
| `sub_manager` | 부책임센터장 | 팀 관리 + 승인 + 초대 |
| `trainer` | 트레이너 | 개인 업무 + 회원 모니터링 |
| `viewer` | 열람 | 대시보드 읽기 전용 |

## 페이지 사이트맵

### 공개 페이지 (로그인 불필요)
| 경로 | 페이지 | 설명 |
|------|--------|------|
| `/` | 랜딩 페이지 | 앱 소개, 로그인 CTA |
| `/join?token=...` | 초대 수락 | 초대 링크 처리 후 대시보드로 이동 |

---

### 📅 일정 관리

| 경로 | 페이지 | 접근 역할 | 주요 기능 |
|------|--------|-----------|-----------|
| `/scheduler` | 스케줄러 | 트레이너 이상 | 주간 캘린더 그리드, 블럭 CRUD, **드래그앤드랍 이동**, 즐겨찾기 블럭, 자동 배치 |
| `/team-schedule` | 팀 스케줄 | 트레이너 이상 | 팀원 통합 주간 뷰, 멤버 필터, private 블럭 마스킹 |
| `/todo` | TO-DO 관리 | 트레이너 이상 | 기간별 TO-DO, 실시간 타이머, 이월(Carry-over), 주간 시간 분배 |

**기능 간 의존 관계:**
- 스케줄러 자동 배치 → TO-DO 목록에서 pending 항목을 읽어 빈 시간에 배치
- 팀 스케줄 → 팀 설정에서 팀이 생성되어야 팀원 목록 표시
- 주별 템플릿 → 스케줄러에서 "템플릿 적용" 버튼으로 해당 주에 일괄 삽입

### 👥 회원 관리

| 경로 | 페이지 | 접근 역할 | 주요 기능 |
|------|--------|-----------|-----------|
| `/members` | 회원 모니터링 | 트레이너 이상 | UID 입력 → 어드민 회원 정보 조회, 역할별 접근 제어 |

**역할별 접근 제어:**
- 트레이너: 본인이 등록한 UID만 조회
- 부책임: 소속 팀의 UID만 조회
- 책임/대표: 전체 조회

**전제 조건:** 어드민 연동(`/admin-settings`)에서 ID/PW 입력 후 세션 유지 필요

## 기능 간 인과 관계 맵

```
[어드민 연동] ──────────────────────────────────────────────────────┐
    ↓ 세션 저장                                                      │
[회원 모니터링] ← UID 조회 → 어드민 API                              │
                                                                    │
[초대 관리] → 초대 링크 생성 → /join?token=...                       │
    ↓ 수락 시 tempoRole 자동 부여                                     │
[시스템 설정] → 직급 변경 → tempoRole 업데이트                        │
    ↓                                                               │
[TempoLayout] → 역할 기반 메뉴 표시/숨김                              │
                                                                    │
[팀 설정] → 팀 생성 → 팀원 추가                                       │
    ↓                                                               │
[팀 스케줄] ← 팀원 스케줄 조회 (private 마스킹)                       │
                                                                    │
[TO-DO 관리] ─────────────────────────────────────────────────────┐ │
    ↓ pending TO-DO                                                │ │
[스케줄러] → 자동 배치 → 빈 시간에 TO-DO 블럭 삽입                    │ │
    ↓ 블럭 완료                                                     │ │
[성장 리포트] ← 타이머 데이터 → Learning Curve 분석                  │ │
                                                                   │ │
[일일 업무 보고] ────────────────────────────────────────────────┘ │ │
    ↓                                                              │ │
[대시보드] ← 보고서 집계 → KPI 차트 ────────────────────────────────┘ │
```

## 향후 구현 예정 (아젠다 C~D)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| TF 프로젝트 | 정규 조직 외 프로젝트 기반 팀 생성, 리소스 분배 비율 추적 | 중 |
| 메신저 Webhook | 카카오톡/슬랙 알림 연동 | 중 |
| 구독 플랜 관리 | SaaS 판매용 플랜 설정 (무료/스탠다드/프리미엄) | 높음 |
| 멀티 센터 지원 | 동일 앱에서 여러 센터 운영 (센터 코드 분리) | 높음 |
| 모바일 앱 | React Native 또는 PWA | 낮음 |
